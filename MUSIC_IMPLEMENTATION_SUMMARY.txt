FAQ
================================================================================

Q: Will this work on mobile?
A: Yes. iOS has autoplay restrictions (requires user gesture), but all
   functionality works. Android has no restrictions.

Q: Does user data get collected?
A: No. All sentiment detection happens client-side. Audio playback is tracked
   only for metrics (which snippets are popular), no user identification.

Q: Can users disable the music?
A: Absolutely. Toggle in settings, respects user preference, persisted.

Q: How much will this impact page load performance?
A: Minimal. Audio files are lazy-loaded (not on initial page load).
   First snippet downloads in background while user reads/interacts.

Q: What if the user has no internet?
A: Music won't play, but app functions normally. Service Worker can cache
   top snippets for offline support (future enhancement).

Q: Will this work on all browsers?
A: Yes. Web Audio API is supported across all modern browsers (Chrome, Firefox,
   Safari, Edge). Graceful degradation if unsupported (app continues without audio).

Q: How do I know it's actually working psychologically?
A: Track metrics pre/post. Look for: longer sessions, more sharing, better
   retention, user feedback. Research shows music + gratitude increases both.
