WHAT WAS FIXED
================================================================================

ISSUE:
  Admin dashboard showed "Low Stock Alert: 0" items
  Inventory management page couldn't filter by stock level

ROOT CAUSE:
  Products API endpoint only returned product data (name, price, image)
  but NOT inventory data (stock counts, thresholds)
  These lived in a separate MongoDB collection that was never queried

SOLUTION:
  Modified app/api/admin/products/route.js to:
  1. Query inventory collection alongside products
  2. Create Map for O(1) productId → inventory lookup
  3. Include stock, lowStockThreshold, lastRestocked in response

IMPACT:
  ✅ /admin - Low Stock Alert now shows real count
  ✅ /admin/inventory - Stock filtering now works
  ✅ /admin/products - Inventory data included
