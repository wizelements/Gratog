## Executive finding

The current email system is not one unified system. It is four disconnected surfaces:

1. `lib/resend-email.js` â€” current transactional Resend sender; writes `email_sends.messageId`.
2. `lib/email/service.js` â€” advanced/queue sender; writes `email_logs.resendId`, but production records show `resendId` is not populated.
3. `app/api/admin/campaigns/send/route.ts` â€” campaign send API; uses `lib/email/service.js`, not `lib/campaign-manager.js`, and does not write `email_sends`.
4. `app/api/webhooks/resend/route.js` â€” webhook endpoint; updates by `email_sends.resendId`, while transactional sends store `messageId`.

Public newsletter signup is currently broken: the UI posts to `/api/newsletter/subscribe`, but that route has no `route.ts`/`route.js`, and production returns `404`.
