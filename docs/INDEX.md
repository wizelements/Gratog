## File Organization Rules

### Audits Folder
- Files with: `AUDIT`, `COMPREHENSIVE`, `DEEP_DIVE`, `FULL_`, `E2E_TESTING`, `ANALYSIS`, `VERIFICATION`, `REPORT`, `SUMMARY`
- Purpose: Analysis & findings

### Fixes Folder
- Files with: `FIX`, `HOTFIX`, `PATCH`, `ISSUE`, `CRITICAL_`, `BUG_`
- Purpose: Solutions & resolutions

### Deployment Folder
- Files with: `DEPLOY`, `VERCEL`, `PRODUCTION`, `READY`, `PUSH`, `WEBHOOK`, `MIGRATION`
- Purpose: Production operations

### Setup Folder
- Files with: `SETUP`, `CONFIG`, `ENVIRONMENT`, `OAUTH`, `CREDENTIALS`, `AUTH`
- Purpose: Initial configuration

### Reference Folder
- Files with: `GUIDE`, `IMPLEMENTATION`, `CHECKLIST`, `ROADMAP`, `ARCHITECTURE`, `COMPLETE_`, `PHASE_`
- Purpose: Ongoing documentation

### Archived Folder
- Everything else that's not actively used
- Purpose: Historical reference
